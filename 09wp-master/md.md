网盘资源导航管理系统

一、项目目标

开发一个包含“后台管理系统 + 前台展示网站”的完整项目。

该系统主要用于管理和展示网盘资源链接，管理员可以在后台添加、编辑、删除资源和分类；普通用户可以在前台浏览、搜索资源，并支持用户注册、登录。



后台管理员登录后，可以管理以下内容：

1.1 网盘资源管理

资源字段包括：

ID
网盘标题
网盘链接
网盘分类ID
网盘分类名称
网盘描述
提取码（可选）
封面图（可选）
标签（可选）
点击量
排序值
状态（显示 / 隐藏）
创建时间
更新时间

后台需要支持：

添加资源
编辑资源
删除资源
资源列表分页
按标题搜索
按分类筛选
按状态筛选
批量删除
批量修改状态
1.2 分类管理

分类字段包括：

ID
分类名称
分类别名/slug
排序值
状态
创建时间
更新时间

后台需要支持：

添加分类
编辑分类
删除分类
分类列表
分类排序
分类状态控制
1.3 用户管理

普通用户字段包括：

ID
用户名
邮箱
手机号（可选）
密码
头像（可选）
状态（正常/禁用）
注册时间
最后登录时间

后台支持：

查看用户列表
搜索用户
禁用/启用用户
删除用户
1.4 管理员登录

使用 GoAdmin 实现后台管理员登录、权限管理、菜单管理。

四、前台网站功能
2.1 首页展示

前台首页展示：

最新资源
热门资源
分类导航
搜索框
2.2 资源列表页

支持：

按分类查看资源
分页展示
显示标题、分类、描述、发布时间、点击量
2.3 资源详情页

展示：

网盘标题
网盘链接
提取码
分类
描述
标签
点击量
发布时间
2.4 搜索功能

前台支持：

按资源标题关键词搜索
模糊搜索
搜索结果分页展示
2.5 用户注册登录

前台普通用户支持：

注册
登录
退出登录
个人中心
修改密码
查看自己收藏的资源（如果你愿意可顺带实现）
五、数据库设计要求

请设计以下核心数据表，并输出建表 SQL：

admins（管理员表，可结合 GoAdmin 管理机制）
users（普通用户表）
categories（资源分类表）
resources（网盘资源表）
user_favorites（用户收藏表，可选但建议实现）
如有必要可增加操作日志表、登录日志表

要求：

字段命名规范
主键自增
常用查询字段建立索引
合理设置默认值
时间字段统一
